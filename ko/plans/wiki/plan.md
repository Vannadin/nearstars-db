# GitHub Wiki 프로그램 — 계획

**목표.** `nearstars-db` GitHub Wiki를 공개용 문서 레이어로 구축합니다.
오너가 정한(2026-07-20) 두 독자층 — 저장소 열람자(이 데이터 엔진이
무엇인지)와 미래 플레이어(모드에 무엇이 들어가는지, 어떻게 설치하는지) —
를 하나로 묶습니다.

**왜 `docs/`를 늘리지 않고 위키인가.** `docs/`는 렌더링된 데이터베이스
뷰어이자 레퍼런스 트리로, 깊고 내부지향적이며 파이프라인을 통해 en/ko로
미러링됩니다. 위키는 얕고 친근한 정문 역할을 합니다. 다섯 페이지로
구성되고, 빌드 단계가 없으며, 데이터 엔진을 건드리지 않고도 편집할 수
있습니다.

## 언어 정책

영어 페이지와 `-ko` 접미사가 붙은 한국어 대응 페이지로 구성합니다
(저장소 관례대로 직역이 아닌 자연스러운 한국어). 모든 페이지 상단에
언어 전환 링크를 둡니다. 영어가 canonical이므로 영어를 먼저 수정한 뒤
`-ko`를 재동기화합니다.

## Canonical 저장소 + 동기화

**위키 git 저장소**(`nearstars-db.wiki.git`)는 최초 초기화 이후 canonical
저장소가 됩니다. 위키 페이지는 이 저장소 안에 중복 저장하지 않습니다
(발산 위험). `plans/wiki/` 세 파일은 결정 사항과 페이지 로스터만
기록합니다. 갱신 흐름은 위키 저장소 클론 → 편집 → 푸시입니다.

## 부트스트랩 제약

GitHub는 위키를 생성하는 API를 제공하지 않으므로, `.wiki.git` 저장소는
웹 UI에서 첫 페이지를 만든 뒤에야 비로소 생성됩니다. 오너가 한 번
(임의의 플레이스홀더 내용으로) 이 작업을 수행하면, 이후 우리가 클론해서
준비된 초안으로 교체하고 푸시합니다.
