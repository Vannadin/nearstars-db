# 50 ly 이내 모든 별 — 풀 마커(큐레이션 없음) — 뷰어 계획

목표는 50 ly 이내 **모든** 별을 **기존 큐레이션 마커와 똑같은 방식으로** 뷰어에
띄우는 것이다(Teff 색 글로우 스프라이트, 광도 비례 크기, 호버 라벨, 클릭 → 정보
패널, 줌인 → 항성 본체). 그래서 사용자가 항성의 **밀도/분포**를 체감하고 이웃
전체를 탐험할 수 있게 한다. 큐레이션 작업이 아니다(Phase 1~3 없음).

이 "필드(field)" 별들은 연구/큐레이션이 필요한 별별 항목만 **빼고는** 큐레이션
마커와 동일하다. 빠지는 항목은 다음과 같다.
- **자전축**(i★/PA 측정값 없음) — 축 자체를 생략한다(기본 축조차 그리지 않는다).
- **항성풍 / 태양권**(바람 데이터 없음) — 거품 없음.
- **공간운동 궤적**(사용자 결정으로 제외; 시선속도도 빠진 경우가 많아 운동이
  신뢰 불가).
- **행성 / 디스크** — 큐레이션된 게 없으므로 시스템 뷰 줌은 맨 별만 보여준다.
기존 큐레이션 시스템 143개는 위 전부를 그대로 유지한다. 필드 별은 그 나머지를
가볍게 채우는 존재다.

## 데이터 소스
- **Gaia Catalogue of Nearby Stars (GCNS; Gaia Collaboration / Smart et al. 2021,
  2021A&A...649A...6G)** — EDR3 기반, 검증됨, 100 pc 이내 ~331k 별; 15.33 pc(= 50 ly)
  이내는 사실상 완전하다. "모든 가까운 별"에 딱 맞는 카탈로그다.
  - 쿼리 대안: `gaiadr3.gaia_source`에서 `parallax > 65.24`(mas) = 50 ly 이내.
- **밝은 별 갭(반드시 처리):** Gaia는 G ≲ 3에서 포화되므로, 가장 밝은 근처 별들 —
  Sirius, Procyon, Altair, Vega, Fomalhaut, α Cen, … — 은 Gaia 측성이 부실하거나
  없다. 그중 여럿이 우리 간판 시스템이다. 그래서 **필드 = Gaia/GCNS + 우리 자체
  큐레이션 별**(밝은 별의 정확한 측성은 이미 갖고 있다)로 둔다. 위치/교차매칭으로
  중복을 제거한다.
- 50 ly 이내 예상 개수: ~2,000~3,000개.

## 무엇을 가져와/구울 것인가 (필드 별별로, 큐레이션 마커가 쓰는 동일 필드)
Gaia에서: `ra, dec, parallax`(→ ly 단위 ICRS xyz), `teff_gspphot`(없으면 `bp_rp`→Teff) →
`teff_to_rgb` 색 + 분광형, `phot_g_mean_mag` + parallax → 절대등급 → 광도 추정 →
마커 크기(동일한 `marker_radius` 모델). 명칭(Designation)은 Gaia/SIMBAD에 통용명이
있으면 그것을, 없으면 Gaia DR3 / 카탈로그 id를 쓴다. pm/rv는 내보내지 않는다(운동
생략). `is_field:true` 플래그를 단 일반 클러스터 객체로, 단일 컴포넌트, 빈 행성으로
내보내 — 기존 빌드 루프가 보통 마커처럼 렌더하게 한다.

## 파이프라인
1. 새 `scripts/viz/fetch_nearby_field.py` → Gaia TAP(parallax > 65.24 mas = ≤50 ly;
   ra/dec/parallax/teff_gspphot/bp_rp/phot_g/source_id 추출) → `db/nearby_field.json`
   (fetch_astrometry처럼 캐시). **SIMBAD 폴백**(fetch_astrometry와 동일): Gaia가
   G ≲ 3에서 포화되므로 가장 밝은 근처 별은 SIMBAD에서 parallax/Teff를 조회한다 —
   SIMBAD-폴백 밝은 별을 필드에 포함해 Sirius/Vega/Arcturus/Capella/Pollux/Altair/
   Procyon/Fomalhaut/α Cen 등이 빠지지 않게 한다. **우리 큐레이션 세트와
   교차매칭-중복제거**(위치/parallax 기준)해서 이미 큐레이션한 별이 두 번 렌더되지
   않게 한다(그것들은 풀 큐레이션 처리로 나오고, 필드는 나머지를 공급한다).
   **갭 체크:** 어느 세트에도 없는 G<4, ≤50 ly 별 → SIMBAD로 추가.
2. `build_starmap.py`가 `db/nearby_field.json`을 읽어 → 큐레이션 클러스터와 나란히
   필드 클러스터(`is_field:true`, spin/wind/motion/planets 없음)를 빌드한다.
3. 뷰어: 필드 클러스터는 **동일한** 경로로 렌더된다. `is_field`가 자전축, 태양권,
   운동 궤적 생성을 차단(gate OUT)한다. 마커 스프라이트 + 호버 라벨 + 클릭 정보 +
   별로 줌은 모두 재사용한다.

## 뷰어 UX
- **"모든 별 / all stars"** 토글(기본 OFF — 마커 ~2,000개를 추가하므로 기본 뷰는
  큐레이션 143개 유지). 켜면 → 50 ly 전체 개체군이 마커로 나타나고, 밀도는 얼마나
  붐비는지에서 바로 읽힌다(사정거리 안의 실제 과밀인 **~47 ly의 Hyades 성단
  가장자리** 포함).
- 색은 Gaia Teff로(큐레이션과 동일한 흑체 맵). 크기는 광도로(동일 모델).
- 라벨: 호버 전용(2,000개 상시 라벨 = 아수라장), 기존 디클러터와 동일.
- 클릭 → 가진 정보로 정보 패널(명칭, 거리, 추정 Teff/분광형). 별 본체 외에 행성/
  컴포넌트는 없음.
- UI/메타에서 정직한 표현: 50 ly 이내 밀도는 거의 균일하다(진짜 은하 구조는 더
  멀다). 바깥으로 가며 약간 옅어지는 것은 관측 완전성(completeness) 때문이지 실제
  밀도가 아니다.

## 성능 (마커 2,000+ 추가)
- 필드 클러스터는 가볍다(궤도/디스크/축/운동/행성 없음 → 해당 배열 전부 비어 있음).
- 기본 off: 노드 숨김, 프레임별 `updateScene` 루프에서 일찍 건너뜀(이미
  `if(!cl.node.visible) return`) → 꺼져 있을 때 비용 0.
- 켜면: 루프에 클러스터 ~2,140개; 필드 별별 경로를 최소로 유지. 프레임별 글로우
  사이징 순회가 너무 무거우면 멀거나 보이지 않는 클러스터를 단락(short-circuit)하거나
  (뷰어 최적화 계획의 P4) 필드 글로우를 배치 처리한다. 최적화 전에 먼저 측정한다.

## 결정 사항 (사용자와 합의)
1. **소스**: 15.33 pc(50 ly) 이내 Gaia GCNS. ✓
2. **색**: Gaia Teff/Bp-Rp 기준(큐레이션과 동일한 흑체 맵). ✓
3. **표현**: 큐레이션과 동일한 풀 마커에서 자전축 / 태양권 / 운동(그리고 큐레이션
   안 된 행성/디스크)만 뺀다. ✓
4. **밝은 별**: 기존 큐레이션 마커로 커버(Gaia가 포화시킴); 필드 = GCNS에서
   큐레이션 중복 제외; G<4 갭 체크. ✓
5. **저장**: `db/nearby_field.json` 캐시 fetch 산출물(astrometry_raw처럼). ✓
6. **기본값**: 토글 뒤에 두고 기본 OFF(권장) — 확인 필요.

## 정직성 / 단서 (UI + 메타에 명기)
- GCNS 완전성은 가장 어두운 M/L 왜성과 분해되지 않은 쌍성(하나로 계수됨)에서
  떨어진다. 가장 밝은 별은 Gaia가 아니라 큐레이션 마커다.
- 필드 별은 운동이 **전혀** 없고(정적), 자전/바람/행성도 없다 — 큐레이션되지 않은,
  시각화 전용이다.
- 색은 보정된 SED가 아니라 지각적 근사다(마커와 동일한 단서).
- 이 별들은 db/systems에 없다(큐레이션 주장 없음). db/nearby_field.json에 산다.

## 검증
- 개수 점검(50 ly 이내 ~2~3k; α Cen / Sirius / Procyon 이중 렌더 안 됨).
- 필드 별은 축/태양권/운동/행성이 없음; 큐레이션 별은 변경 없음.
- 재현 가능한 빌드; payload 크기 기록; node --check; check.sh green.
- 비주얼: 맵 스케일에서 토글 켜기 → 밀도(Hyades 가장자리 포함)가 읽힘; 필드 별로
  줌인 → 오버레이 없는 맨 별; 부동소수점 원점(floating-origin) 리베이스 안정;
  성능 허용 범위.

## 범위 / 비목표
- 필드 별에 큐레이션, 행성, 자전/바람/운동 없음.
- 단일 파일 뷰어, CDN importmap, GitHub-Pages 모델 변경 없음.
- 새 fetch + 빌더 emit + `is_field` 게이팅 외에 DB/파이프라인 변경 없음.
