# 상대성이론 모드 — 아광속 특수상대론 게임플레이 레이어

**NearStars 연결점.** 성간 이동은 RP-1 완주 후 엔드게임 확장이다
(`project_nearstars_interstellar_expansion` 참조). 게이트0 타당성 검토에서
반응추진으로는 상대론적 순항에 빠르게 도달할 수 없고 워프 모드는 Principia와
비호환이라는 결론이 났고, 그래서 방향은 *게임플레이 우선*으로 잡혔다. 이 노트는
초광속 치팅 없이 **광속의 벽을 체감시키는** 특수상대론 게임플레이 레이어를
스코핑한다. 즉 빛의 속도에 다가갈수록 실제로 배를 가속시키는 추력이 사라지고,
대신 승무원의 고유시간 기준 물자 소모가 느려진다. 보류해 둔 시간 팽창(시계)
메커닉의 아광속 짝이다.

**범위.**
- **포함** — 코어 메커닉(추력 스케일링·물자 스케일링·계기판) 설계와 옵셔널
  3티어 시각 레이어. 기준계 선택. Principia/SigmaBinary 호환 논리. 물리 그라운딩.
- **제외** — 실제 시계 시간 팽창 / 타임워프 조작(추후 "시간" 요소로 보류).
  에너지·연료 질량비 폭증 경제학. 일반상대론 효과(이건 특수상대론 한정).
- **이 노트에서 코드 변경 없음.** 스코핑 문서다. 구현은 C# / 셰이더 플러그인이고
  슐츠 담당이다 (`project_nearstars_mod_plugins_schultz`).

**외부 레퍼런스.**
- Principia (뉴턴 n-body 적분기, 바리센터 관성계):
  https://github.com/mockingbirdnest/Principia
- Singularity (KSP 블랙홀 중력렌즈 스크린 스페이스 포스트프로세스 — KSP에서
  상대론적 광행차 셰이더 파이프라인이 가능하다는 증거). repo URL은 핸드오프 전 확인.
- Kerbalism (시간 기반 물자·생명유지 소모 — 고유시간 물자 스케일링의 훅).

## 2. 본문

### 2.1 코어 메커닉 — 추력 / γ³

운동 방향으로 힘을 줄 때 특수상대론은 다음을 준다.

```
a = F / (γ³ · m)        γ = 1 / √(1 − β²),   β = v/c
```

따라서 실제로 배를 가속시키는 추력은 **F_eff = F_정격 / γ³**다. 이게 "종방향
질량" γ³m이다. 이걸 적용하면 광속이 자동 점근벽이 된다 — β → 1이면 γ³가 폭발해서
정격 추력이 아무리 커도 가속 → 0. 인위적 최대속도 cap이 필요 없다.

적분기 관련 동치성 — `a = F/(γ³m)`는 "정상 질량에 약한 힘"으로 보든 "정상 힘에
γ³배 무거운 질량"으로 보든 동일하다. 뉴턴 적분기에 *약해진* 힘을 먹이면 종방향
상대론 질량 효과가 정확히 재현된다.

| β | γ³ | 유효 추력 |
|---|----|-----------|
| 0.1 | 1.015 | 98.5% |
| 0.5 | 1.54 | 65% |
| 0.9 | 12.1 | 8% |
| 0.99 | 356 | 0.3% |

커브는 ~0.5c 아래에선 완만하다 광속 근처에서 가파르게 꺾인다 — 의도한 "빛의 벽을
미는" 느낌이다. 기본은 순수 1/γ³로 두고, 게임필을 위해 더 일찍 켜고 싶으면 튜너를
노출한다.

### 2.2 보상 메커닉 — 물자 소모 / γ

고유시간은 `dτ = dt / γ`로 흐른다. 고유시간에 묶인 소모를 **1/γ**로 스케일하면
물리적으로 정확하고 시계나 타임워프는 전혀 안 건드린다(시간 요소 보류 유지).
추력 페널티와 부호가 반대라, 이게 메커닉을 평면적 세금이 아니라 진짜 트레이드로
만든다.

| β | γ | 유효 추력 (×1/γ³) | 물자 소모 (×1/γ) |
|---|---|-------------------|-------------------|
| 0.5 | 1.15 | 65% | 87% |
| 0.9 | 2.29 | 8% | 44% |
| 0.99 | 7.09 | 0.3% | 14% |

→ 가속은 짓눌리지만, 이미 빠른 배는 승무원을 훨씬 오래 살린다. 그래서 플레이어가
0.9c 순항을 의도적으로 노릴 전략적 이유가 생긴다. Kerbalism의 시간 기반 소모
모델이 자연스러운 훅이다.

**고유시간에 묶인 소모만 스케일한다** — 생명유지, 승무원 식량·산소, 샘플 변질.
**엔진 추진제는 제외** — 그건 힘을 만드느라 좌표시간으로 탄다. 이것까지 스케일하면
이중 보상("연료도 천천히 *그리고* 빠르게")이 된다.

### 2.3 기준계 — 태양계 바리센터

KSP 속도는 기준 천체 상대값(궤도/표면)이라 성간 비행에선 애매하다. 여정 전체에
**출발 시점 태양계 바리센터 관성계 고정**을 쓴다. 행성계 내부에선 β가 미미해서
사실상 성간 순항에서만 메커닉이 켜진다.

성간 뉘앙스 — 다른 별들은 태양 바리센터에 대해 고유운동을 갖지만(α Cen ≈ 22 km/s
≈ 7×10⁻⁵ c) 무시 가능하므로, "목적지에서 정지"해도 β ≈ 0으로 읽힌다. 여정 전체에
단일 고정 관성계가 가장 일관적이다.

이건 Principia와 맞물린다 — Principia는 이미 바리센터 관성계로 적분하고 바리센터
속도를 직접 넘겨줄 수 있다. 스톡 프로파일에선 우리가 직접 바리센터 속도를 계산해야
한다(스톡은 SOI 상대속도만 준다).

### 2.4 Principia / SigmaBinary 호환

앞선 "Principia 비호환" 판정은 **워프/FTL** 얘기였지 이게 아니다. 이 메커닉은
정반대 경우다.

- Principia는 뉴턴 n-body 중력을 적분하고 비중력 힘(엔진)은 게임에서 받아온다.
  우리는 엔진 힘을 *적용되기 직전에* 줄인다(× 1/γ³). Principia 입장에선 그냥
  약한 분사로 보인다 — 적분기는 무손상.
- 물자 스케일링은 Kerbalism 소모율 변경 — 역시 적분기 밖.
- 미세한 철학적 불일치는 있다(Principia가 운동량을 뉴턴식으로 봄). 하지만 여기
  모든 SR 효과는 **힘·소모율 변조 레이어**라 적분기와 안 싸운다. 시공간을 안 비튼다.

결론 — 이 레이어는 **양쪽** 프로파일(Principia n-body·SigmaBinary)에 다 얹힌다.
슐츠와 확인할 한 가지는 Principia가 적분 전 프레임당 고유 힘을 수정할 깔끔한 훅을
노출하는지다.

### 2.5 옵셔널 시각 레이어 — 3티어

Singularity가 KSP에서 상대론적 스크린 스페이스 포스트프로세스가 가능함을 증명한다
(블랙홀 렌즈를 이미 이 방식으로 한다). 상대론적 룩은 물리적으로 정확한 세 성분으로
쪼개진다.

| 성분 | 변환 | 화면 |
|------|------|------|
| 광행차 | cos θ′ = (cos θ − β)/(1 − β cos θ) | 별들이 진행 방향으로 뭉침 (전방 헤드라이트) |
| 도플러 | D = 1/[γ(1 − β cos θ)] | 전방 청색편이·후방 적색편이 (각도별 색) |
| 빔잉 | 세기 ∝ D⁴ (bolometric, 비편광 I_ν ∝ D³) | 전방 밝아지고 후방 어두워짐 |

이 셋이 합쳐진 게 "스타보우"다. 각 픽셀에 시선각 θ와 β만 있으면 되니 셰이더 한
패스로 떨어진다.

**시각은 메커닉과 분리한다** — 셰이더가 늦거나 옵셔널이어도 게임플레이는 멀쩡하게.

- **Tier 0 — 코어, 시각 없음.** 추력/γ³ + 물자/γ + 계기판. 이것만으로 게임플레이
  완결.
- **Tier 1 — 싼 시각.** 도플러 색조 + 빔잉 밝기만, 기하 왜곡 없음. 가성비 최고,
  ~0.5c 넘기면 "빠르게 가고 있다"를 판다.
- **Tier 2 — 풀 스타보우.** 광행차 기하 왜곡 추가. Singularity급 난이도지만 증명된
  길.

이 레이어는 스크린 스페이스 포스트프로세스 — Scatterer와 같은 계열 — 라 기존 시각
파이프라인에 인접한다.

## 3. NearStars 함의

- 이건 **성간 엔드게임용 게임플레이 레이어**지 물리 적분기가 아니다. 게이트0의
  "게임플레이 우선" 결정과 일관되고, 워프와 달리 Principia 안전하다.
- **권장 빌드 순서** — Tier 0 메커닉 먼저(추력 + 물자 + 계기판) → 자체로 출시
  가능하고 독립적 → 시각은 나중에, Tier 2보다 Tier 1 먼저.
- **핸드오프** — 이 노트가 슐츠용 브리프다(`project_nearstars_mod_plugins_schultz`).
  플러그인 세 조각 — (a) 프레임당 고유 힘 1/γ³ 스케일링, (b) Kerbalism 고유시간 소모
  1/γ 스케일링, (c) 포스트프로세스 셰이더(Tier 1–2). 이 레이어는 NearStars DB / cfg
  델타를 함의하지 않는다.

  확인.)
- 스톡 프로파일 — Principia 없이 프레임마다 바리센터 속도를 계산하는 가장 싼 정확한
  방법.
- 온셋 튜닝 — 순수 1/γ³ 유지냐, 더 일찍 체감시키게 β-스케일 / 지수 튜너를 노출하냐.
  기본 = 순수, 튜너 옵셔널.
- 고유시간에 묶인 것으로 취급할 물자 정확한 목록(생명유지 / 승무원 소모재 / 과학
  샘플 변질) vs 좌표시간(추진제, 리액션휠 등).
- 다른 별 도착 시 출발 태양 바리센터 기준계를 유지하느냐(권장) 목적지 바리센터로
  재기준하느냐. 고유운동은 어느 쪽이든 무시 가능.
- 시각 포스트프로세스가 Scatterer / Singularity와 함께 설치됐을 때 깔끔하게 합성되는가
  (셰이더 패스 순서)?
