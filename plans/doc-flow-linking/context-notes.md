Owner ask: make inter-document flow natural across three axes - Phase 1-4 flow,

### Reconnaissance findings (why the work is shaped this way)
- Phase 3 -> Phase 4 links: 0 (either direction). Phase 4 boards link out only to
  DB index + paper refs.
- reports hub covers Phase 2 + Phase 3, omits Phase 4 and methodology-index.
- Basis-column -> specific methodology doc links near-absent; topic methodology
  docs are effectively reachable only via methodology-index.
- `build_html.py:207` emits markdown links verbatim, so phase-3 `.md` Related
  targets render as `.md` hrefs -> broken on the published site. Real bug; fixing
  it is a prerequisite for the flow work.

### Key design decision: slug-match, not a yaml field
Phase 4 board body slugs mostly equal Phase 3 slugs (40-eridani-*, proxima-cen-b/c/d,
tau-cet, barnards-star, fomalhaut, alpha-centauri-{a,b,a-b}). Art-name bodies
(pandora, cassandra, chaos, dante, hades, polyphemus; proxima-cen-c-i moon) have
no Phase 3 counterpart. So links are generated by slug existence check - no new
yaml field, no board data change (also avoids colliding with the parallel session
that edits alpha_centauri.yaml).
- Phase 3 -> Phase 4: emit when `docs/phase4/*/<slug>.html` exists (glob at build).
- Phase 4 -> Phase 3: emit when `docs/phase3/<body_slug>.md` source exists (stable).

### Reference/plans render target
docs/reference/*.md and plans/*.md render into the doc-hub at
`docs/wiki/reference__<slug>.html` / `plans__<slug>.html` (build_docs.py `_LINK_MAP`).
The phase-3 `.md`->`.html` rewrite must map reference/plans links there, not to a
non-existent same-dir `.html`.
