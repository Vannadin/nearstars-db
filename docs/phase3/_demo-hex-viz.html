설계 메모

왜 perceived 만 표시하는가

합성 hex 는 "플레이어가 보는 색" (모항성 조명 반영). KSP cfg 의 텍스처 색은 이론적으로 "조명 곱하기 전" 이어야 하지만 — (1) 합성 픽 자체가 곱셈 모델로 유도된 게 아니라 정성 큐레이션, (2) KSP 셰이더 파이프라인 (Kopernicus PQS / Scatterer 톤매핑 / EVE 알파 합성) 이 단순 곱하기가 아니라 결국 인게임 튠 필수. 따라서 Phase 3 는 perceived 한 종류만 명시하고, cfg-asset 변환은 downstream cfg writer 가 책임.

오로라 스펙트럼 그라디언트

오로라는 self-emissive — 별빛과 무관. 단일 hex 보다 발광 종의 실제 파장 범위가 더 정보가 많아서 그라디언트로 표시. Bruton wavelength→RGB 근사 (380–780 nm). 원자선 ([OI] 557.7 nm 등) 은 ±20 nm 범위 + 흰 마커, 분자 밴드 (CO₂⁺ FDB 580–700 nm 등) 는 자연스러운 범위. 파장 정보가 없는 케이스 (e.g. Mie-scattered UV) 는 정상 hex 칩으로 폴백.

build_html.py 후처리 패스

합성 마크다운 → HTML 변환 시 자동: (1) Decisions 표 감지 → 표 위에 palette band, (2) 모든 `#xxxxxx` → 칩 + 코드, (3) aurora_color_* 행의 Value 셀은 Basis 의 nm 표기 파싱 → spectrum bar. 7행성 일괄.
