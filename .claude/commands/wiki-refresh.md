Scan all `.md` files in the wiki tiers and ingest any that lack YAML frontmatter.

Procedure:

1. Find all `.md` files in: `docs/reference/`, `docs/phase3/` (excluding `_papers/` and `_bib/`), `plans/`, `phase2/<topic>/`, `phase3/<system>/`, `.agents/skills/*/` (skill SKILL.md + references where appropriate), `docs/wiki/`, `AGENTS.md`, `README.md`, `docs/phase1-50ly/`, `docs/famous-20-non-hosts/`.

2. Filter out files that ALREADY have YAML frontmatter (`head -1 == "---"`).

3. For each remaining file, run the `/wiki-ingest` workflow (delegate to SKILL.md §4).

4. After all ingests complete, run `/wiki-lint` and append both ingest entries + a final summary to `docs/wiki/log.md`.

5. Update `docs/wiki/index.md` cluster sections and `docs/wiki/hot.md`.

Skip:
- Raw tier (`docs/phase3/_papers/`, `db/`, etc.) — never modified.
- `ko/` mirrors — regenerated by user from English source, don't auto-edit.
- Local-only skill workspaces (`scatterer-cfg-workspace/`, `eve-cfg-workspace/`, etc.) — ingest IS allowed there but output stays gitignored (same procedure, different tier).
- `dist/`, `.venv/`, `__pycache__/`, `.git/`.

Report at the end: total files scanned, ingested count, skipped count (with reasons), any lint findings. Under 250 words.
